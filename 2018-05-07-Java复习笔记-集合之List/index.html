介绍

  1. List是有序的对象集合。

  2. List接口表示Collections Framework中的列表。

  3. List可以包含重复的元素。您还可以在列表中存储多个空值。
  4. List接口继承Collection接口。它添加了一些方法来支持使用索引访问List的元素。
  5. 它允许您将元素添加到List的末尾或由称为索引的整数标识的任何位置。
  6. List中元素的索引从零开始。

列表类型

以下是List接口的许多实现类中的两个：

  • ArrayList - 由数组实现
  • LinkedList - 由链表实现

如果经常获取和设置列表的元素，则ArrayList的性能会更好。

访问ArrayList中的元素更快。

添加或删除ArrayList中的元素执行速度较慢，除非从最后完成。

LinkedList在列表中间添加和删除元素的效果更好。

访问列表元素的速度较慢，除非在列表的头部。
