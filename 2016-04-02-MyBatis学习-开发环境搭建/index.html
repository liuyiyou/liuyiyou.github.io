```java
public class UserMapperTest {
@Before
public void setUp() {
DBUtils.initHSQLData(“chapter-01.sql”);
}
@Test
public void testInsertUser() {
int result= templae(sqlsession->sqlsession.insert(“cn.liuyiyou.mybatis.mapper.chapter01.UserMapper.insertUser”));
Assert.assertEquals(1, result);
}
}
如果是使用在idea中，在默认配置下，如果xml映射文件放在和Mapper同级下可能会出现错误，报找不到资源文件错误，这是idea编辑决定的，通过看编译后的文件会发现，编译的时候只编译resources中的xml文件，所以最好放到resources下，如果不可以，则在resources下建立与Mapper同名目录，再放入
必须有session.commit()。而且最好写在Assert之后。因为如果不这句。事务不会提交。写在后面的话，如果和预期结果不同的话，也会提交事务
