Docker环境信息

容器生命周期管理

-i : 表示使用交互模式，始终保持输入流开放
-t ：表示分配一个伪终端，一般两个参数结合使用-it，即可在容器中利用打开的伪终端进行交互操作
– name： 指定docker run命令启动容器的名字，若无此选项，Docker将为容器随机分配一个名字
-p ： 将容器端口暴露给宿主机的端口，格式为 hostPort:container-port
-v: 挂载一个Volume，格式为 [host-dir]:[contaner-dir]:[rw|ro]
– restart : awalys

docker run

-i 标识保证容器中stdin是开启的
-t 标识Docker为要创建的容器分配一个tty终端
-d 标识是让 docker 容器在后台运行,并返回容器ID。
-p 标识通知 Docker 将容器内部使用的网络端口映射到我们使用的主机上。
–name 定义一个容器的名字，如果在执行docker run时没有指定Name，那么deamon会自动生成一个随机数字符串当做UUID。
-m :设置容器使用内存最大值；

sh
docker run –name mysql -p 6033:3306 -e MYSQL_ROOT_PASSWORD=123456 -dmysql/mysql-server:latest

docker run –name redmine -p 9003:80 -p 9023:22 -d -v /var/redmine/files:/redmine/files -v /var/redmine/mysql:/var/lib/mysql sameersbn/redmine
`
