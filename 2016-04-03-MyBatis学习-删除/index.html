MyBatis学习-删除

发表于 2016-04-03 | 分类于 mybatis | 阅读次数:
字数统计: 1,241 | 阅读时长 ≈ 6

MyBatis最复杂的是其查询，其次是插入、更新、和删除。本着由难到易的原则，先来练习删除

使用sqlsession来进行删除的时候，只有两个重载的方法，int delete(String statement) ,
int delete(String statement, Object parameter)。两者均返回受影响的条数，底层使用的是update方法（这个在后面会有说明）

本实例包含以下实例：

  • 没有参数
  • 根据主键删除（一个参数）
  • 根据名字删除（一个参数，非主键）
  • 根据实体类删除（一个参数，实体类）
  • 根据Map删除（一个参数，是Map类型）
  • 批量删除（一个参数，参数是数组。使用了动态SQL）
  • 批量删除（一个参数，参数是List）

动态SQL说明

foreach：类似java中的for循环，

collection：表示java传递过来的参数类型。其中collection=”array”表示传递过来的参数是一个数组。collection=”list”表示传递过来的是一个list列表。如果
传递过来的是数组，而用list接受的话，会报错。

items：要循环的数组或者列表。值是什么无所谓。只要和#{ids}前后保持一致即可。否则会报错org.apache.ibatis.binding.BindingException: Parameter ‘id’ not found. Available parameters are [list]。

open：表示拼接SQL开始的时候，用什么，
close：则表示循环结束后用什么。separator则是循环之间用什么分割。

比如后台传递过来的是 new Integer[]{1,2}。则拼接的sql语句则为delete from user where id in (-open id[0]=1~? ,-separtor id[1]2~? )-close。

注意事项

在UserMapper中需要注意：parameterType可以不写，MyBatis会自动判断。如果写了的话，对于基本类型，则要写对。比如需要int，传递过来的却是string。则会报错。而对于非基本类型，则没关系。比如在id=deleteByEntity中，如果传递过来一个Map。也能进行正常的删除
如果传递一个null过来。则需要注意，因为这个时候的sql语句是where age = null 。而在mysql中，只有where age is null 才会被识别的。所以如果要根据null来查询的是，需要注意
