多线程和多进程有什么区别?

本质区别在于每个进程有它自己的完备集。线程则共享相同的数据。

线程的实现方式(具体参看本目录：线程启动的方式)

  1. 实现Runnable接口，并通过new Thread(runnable)，将实现该接口的类传入
  2. 继承Thread。不建议，应该尽量从机制上减少需要并行运行的任务数。可以使用线程池

不要调用Thread类或Runnable对象的run方法。直接调用run方法只会在当前线程中执行任务，并不会启动新的线程，正确的做法是调用thrad.start方法。它会创建一个新的线程来执行run方法。

中断线程的方法

Thread.currentThread().isInterrupted()
当然，如果一个线程被阻塞了，就无法检查中断状态了。会产生一个InterruptException异常

线程状态或者生命周期：

  1. New（新生的）： new的时候
  2. Runnable （可运行）：调用start的时候，当然可运行并不表示正在运行
  3. Blocked（被阻塞）
  4. Dead（死亡）

线程在什么时候进入阻塞状态：

1. 线程通过调用sleep方法进入睡眠状态
2. 线程调用一个在I/O上被阻塞的操作，即该操作在输入输出操作完成之前不会返回到它的调用者
3. 线程试图得到一个锁，而该锁正被其他线程持有。
4. 线程在等待某个触发条件
5. 有人调用了线程的suspend方法。

阻塞状态下的怎样回到运行状态

1. 线程被设置于睡眠状态，而且已经经过指定的毫秒数
2. 线程正在等待I/O操作，而且该操作已经完成
3. 正在等待另一个线程所持有锁，而且另一个线程已经释放了该锁的所有权
4. 线程正在等待某个触发条件，而且另一个线程发出了信号表明条件已经发生了变化
5. 线程已经被挂起，而且有人调用了resume方法。不过resume方法已经废弃

线程的死亡

1. 因为run方法正常推辞自然死亡
2. 以为一个未不活的异常终止了run方法而使线程衰死
• 线程属性： 线程优先级，与操作系统有关。
• 线程组 ThreadGroup：统一管理整个群组的线程
• 未捕获异常处理器

线程的run方法不能抛出任何被检查的异常，但是不被检查的异常可以导致线程终止，如果出现这样的情况，线程就死亡了。然而并不需要任何catch子句来处理可以被传播的异常，在线程死亡之前，异常将被传递给未捕获异常的处理器来处理。处理器必须实现Thread.UncaughtExceptionHandler接口的类
• 如何加入线程
使用join方法加入线程，则那个线程会等待到该线程加入后才继续工作
