Error和Exception的区别

Error表示系统内部错误或者资源耗尽，一般是硬件或者非程序相关的问题，程序不需要处理

Exception：分两种：RuntimeException及其子类，不需要显示关注，比如NPE之类，其他Exception，需要显示处理

Exception和RuntimeException的区别

Exception是受检查异常，即如果抛出该异常，调用方必须处理或者继续抛出
RunTimeException：非检查异常，抛出该异常后，调用方不需要进行处理。一般采用此方法，除非某些时刻必须显示处理的时候，才使用第一种

RuntimeException：类型转换、数组越界、NPE

throw和throws

throw用在方法体中，用来抛出异常

异常处理方法

  1. 在Controller层使用try-catch进行处理
  2. 直接抛给上层框架进行统一处理
  3. 捕获异常需要占用时间比较多，因此只有在异常情况下使用异常机制
  4. 不过分的细化异常
  5. 利用异常层次机构
  6. 不要压制异常
  7. 早抛出，晚捕获
    8.try-catch的时候，一定要在catch中做处理，不要留空，不然虽然抛出了异常，但是日志里面没有
